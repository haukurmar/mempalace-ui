# web

The MemPalace local web UI — a TanStack Start app that runs on your machine and reads the
local palace at `~/.mempalace/palace/` via server functions.

## Local dev

```sh
pnpm --filter web dev    # http://127.0.0.1:3000
```

The dev server explicitly binds to `127.0.0.1` (see "Local dev" below); using
`http://127.0.0.1:3000` matches that binding exactly. `http://localhost:3000`
also works on most setups, but on machines where `localhost` resolves to IPv6
`::1` first the connection can stall — prefer the literal `127.0.0.1` form.

Override the port:

```sh
MEMPAL_UI_PORT=4000 pnpm --filter web dev
```

The dev server binds to `127.0.0.1` only — this is a single-user developer tool and is
deliberately not reachable from the LAN.

## Phase 7 scope

This is the bare scaffold. It deliberately ships:

- TanStack Start + React 19 + Tailwind v4 wired through the `@memui/design-tokens` preset.
- File-based routing via TanStack Router (`routeTree.gen.ts` is auto-generated by the
  Vite plugin on first build / dev start).
- Root layout with `Toaster` (sonner) and a hardcoded `data-density="comfortable"` on
  `<html>`. Phase 15 wires the localStorage source.
- A placeholder `/` route that demonstrates `EmptyState`, `Button`, and a working
  `CommandBar` (its built-in Cmd+K listener handles toggling — the on-page button is
  just an alternate entry point).
- A `QueryClient` with project-wide defaults (`staleTime: 30_000`, no window-focus
  refetch, single retry) wired into the router context via
  `setupRouterSsrQueryIntegration`.

Deliberately deferred:

- Server functions and palace connection (Phase 8).
- Keyboard registry root + cheatsheet (Phase 11). `CommandBar` works standalone today.
- Real palace data; the placeholder actions are stubs.
